📖 Argus User Guide & Recruiter Manual

Recruiter Portal: This guide provides a summary of the technical skills demonstrated in this project, step-by-step verification procedures, and details on how simulated fallbacks keep this platform operational in containerized cloud environments.

Developer Skill Showcase

Argus is built to demonstrate production-grade software engineering, network protocol understanding, security heuristics, and modern UI/UX design. Key skills showcased include:

⚙️ Concurrent Socket Programming The Port Scanner uses Python's socket library and concurrent.futures thread pools to scan hundreds of ports concurrently with service fingerprinting.
📡 Raw Packet Analysis Integrates Scapy to sniff raw network interfaces, dissect OSI layer layers (IP/TCP/UDP/Raw), and decode packet payloads for inspection.
🔒 Session & Credential Security Implements custom session token checkers, secure password hashing (SHA-256), route protection decorators, and brute force login lockouts.
⚡ Regex Parsing & Heuristics Uses complex regular expressions to search text bodies or crawl HTML responses, extracting and categorizing IPv4/IPv6, TLDs, email hashes, and malware vectors.

Autonomous Simulation Engine

When Argus is hosted inside cloud containers (such as **Hugging Face Spaces**), low-level system hooks are restricted:

  • The Windows Event Log parser raises dependency errors because the container is running Linux.
  • Raw packet sniffing using Scapy raises permission errors because cloud nodes block root privilege access.
How Argus Solves This: Argus includes a **real-time simulation loop**. When starting a packet capture or scanning logs, if the backend detects that low-level access is unavailable, it automatically launches background daemon threads to generate realistic security telemetry. This guarantees that charts, timelines, feeds, and alert notifications remain 100% operational for reviewers.

A status badge indicating Simulation or Live SOC is displayed at the top of the navigation bar to inform the user of the system's operational status.
